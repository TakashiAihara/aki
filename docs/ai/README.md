## 🎯 クイックスタート

### 初めて AI 開発を始める方

1. **[modern-ai.md](../development/modern-ai.md)** を読む
   - Phase 0-3 のワークフローを理解
   - 各 AI ツールの役割分担を把握

2. **[プロンプト規約](prompt-guidelines.md)** を読む
   - 基本的な指示の出し方を学ぶ

3. **フェーズ別プロンプト集** を参照
   - Phase 0: [設計フェーズ](phase-0-design-prompts.md)
   - Phase 1: [実装フェーズ](phase-1-implementation-prompts.md)

4. **[アンチパターン集](anti-patterns.md)** で失敗例を確認
   - よくある間違いを避ける

### Phase 0（設計）を始める方

```
1. Phase 0 用プロンプト集を開く
2. 要件整理テンプレートをコピー
3. Claude Code に貼り付けて対話開始
4. ADR 形式で設計を記録
```

### Phase 1（実装）を始める方

```
1. Phase 0 で作成した ADR / Spec を確認
2. Phase 1 用プロンプト集を開く
3. 実装タスクに応じたテンプレートを選択
4. ADR への参照を含めて AI に指示
```

## 🔑 重要な原則

### 1. 制約条件を最初に与える

❌ **Bad**: 「ユーザー管理機能を作って」

✅ **Good**: 「ユーザー管理機能を作って。制約: YAGNI、最小構成、将来拡張は考えない」

### 2. フェーズを明確にする

❌ **Bad**: いきなり実装を依頼

✅ **Good**: Phase 0 で設計を固めてから Phase 1 で実装

### 3. ADR / Spec を参照させる

❌ **Bad**: 「良い感じに実装して」

✅ **Good**: 「ADR-0001 の設計に従って実装して」

### 4. 学習利用を意識する

- **Phase 0**: 学習利用されない AI（Claude Code）で業務思想を固定
- **Phase 1**: 学習利用される AI でも OK（Boilerplate のみ生成）

## 💡 よくある質問

### Q: Phase 0 をスキップしてすぐ実装しても良い?

A: **NO**。Phase 0 で業務ロジックの思想を固定しないと、Phase 1 で AI が過剰設計します。

### Q: 全てのタスクで ADR を作る必要がある?

A: **NO**。変更コストが高い決定や複数の選択肢がある場合のみ ADR を作成してください。

### Q: プロンプトは英語で書くべき？日本語で書くべき？

A: **日本語推奨**。SiliconFlow（Qwen）は日本語性能が高いです。ただし、英語の方が精度が高いモデルもあるため、ツールに応じて使い分けてください。

### Q: 学習利用される AI に何を渡してはいけない?

A: 業務ロジックの **思想・制約・独自パターン** は Phase 0 で分離してください。Boilerplate / CRUD は Phase 1 で渡しても問題ありません。
